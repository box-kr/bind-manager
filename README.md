## 📥 설치 방법 (Homebrew)

Bind Manager는 Homebrew Cask를 통해 간편하게 설치하고 업데이트할 수 있습니다. 터미널(Terminal)을 열고 아래 명령어를 순서대로 입력하세요.

### 1단계: 저장소 추가 (Tap)
```bash
brew tap box-kr/homebrew-bind-manager
```

### 2단계: 앱 설치
```bash
brew install --cask bind-manager
```

### 업데이트 방법
설치 후 새로운 버전이 출시되었을 경우, 아래 명령어를 통해 최신 버전으로 업데이트할 수 있습니다. (앱 내 자동 업데이트 기능을 통해서도 백그라운드로 안전하게 업데이트를 수행할 수 있습니다.)
```bash
brew upgrade bind-manager
```

## ✨ 주요 기능

- **🔍 실시간 포트 모니터링**: 현재 사용 중인 로컬 포트, 프로토콜(TCP/UDP), IP 주소 및 매핑 정보를 실시간으로 감지하고 트리 형태로 명확히 표시합니다.
- **🐳 Docker 완벽 통합 지원**:
  - 호스트 머신의 포트를 점유하고 있는 Docker 컨테이너를 자동으로 식별하여 `CONTAINER` 뱃지와 함께 보여줍니다.
  - 컨테이너 내부 포트와 호스트 포트 간의 포트포워딩(매핑) 정보를 직관적으로 제공합니다.
  - 클릭 한 번으로 해당 컨테이너의 실시간 **Log**를 조회하거나 **Shell** 터미널을 직접 엽니다.
  - 불필요한 컨테이너를 앱 내에서 즉시 **Stop** 시킬 수 있습니다.
- **🖥️ 프로세스 및 앱 상세 정보 제공**:
  - 포트를 사용 중인 일반 데스크탑 앱이나 백그라운드 프로세스의 PID, 앱 이름, 창 제목(Window Title), 전체 실행 경로(Path)를 한눈에 파악할 수 있습니다.
  - 시스템 리소스를 낭비하거나 포트 충돌을 일으키는 타겟 프로세스를 즉시 **Kill (강제 종료)** 할 수 있습니다.
- **🔎 강력한 다중 조건 검색**: 포트 번호(예: `8080`), 앱 이름(예: `node`), 컨테이너 ID 또는 실행 경로 등 원하는 키워드로 활성화된 항목을 빠르게 필터링할 수 있습니다.
- **🔄 자동 업데이트 감지**: 앱 실행 시 GitHub의 최신 릴리즈 버전을 확인하고, 새로운 버전이 배포되었을 때 Homebrew를 통해 자동 업데이트를 진행할 수 있는 알림을 제공합니다.

## 📝 사용 가이드

앱을 실행하면 현재 네트워크 포트를 사용 중인 프로세스와 컨테이너 목록이 직관적인 트리(Tree) 구조로 표시됩니다.

1. **상태 확인**: 화면 리스트에서 파란색(`APP`), 주황색(`PROCESS`), 하늘색(`CONTAINER`) 뱃지를 통해 포트를 점유한 타겟의 종류를 쉽게 구분할 수 있습니다.
2. **프로세스 관리 (Kill)**: 특정 포트를 점유하여 다른 서비스 실행에 방해가 되는 프로세스를 찾았다면, 우측의 `Kill` 버튼을 눌러 즉시 강제 종료(`kill -9`) 할 수 있습니다.
3. **Docker 컨테이너 제어**: 🐳 아이콘으로 표시된 Docker 항목에서 `Log` 버튼을 눌러 실시간 컨테이너 로그를 보거나, `Shell` 버튼으로 터미널에 접속하고, `Stop` 버튼으로 컨테이너를 중지하세요.
4. **검색 활용**: 상단의 검색창에 포트 번호, 프로그램 이름, 혹은 경로의 일부를 입력하면 해당하는 포트나 컨테이너만 즉각적으로 필터링해서 보여줍니다.
5. **새로고침 및 앱 정보**: 상단 툴바의 `새로고침(🔄)` 버튼으로 최신 상태를 반영하거나, `정보(ℹ️)` 버튼을 눌러 현재 앱 버전을 확인하고 수동으로 업데이트를 체크할 수 있습니다.

## 🛒 최근 업데이트 & 릴리즈 노트

**v1.0.1 (네트워크 요약 정보 UI 추가)**
- **메인 화면 하단 Footer 신설**: 현재 시스템의 네트워크 상태를 요약해서 보여주는 상태바가 화면 하단에 추가되었습니다.
  - **전체 Listen 개수**: 현재 모니터링 중인 전체 포트 개수를 한눈에 파악할 수 있습니다.
  - **Local IP**: 현재 기기가 연결된 내부 네트워크의 로컬 IP 주소(`en0`, `en1` 기준)를 자동으로 감지하여 표시합니다.
  - **NAT IP**: 외부에서 식별되는 공인 IP(NAT IP)를 API(`api.myip.com`)를 통해 비동기로 조회하여 표시합니다.
  - **IP 새로고침**: 우측의 새로고침 버튼을 눌러 IP 정보를 즉각적으로 다시 불러올 수 있습니다.

## ❗️ 문제 해결 및 참고 사항

**"손상된 앱이기 때문에 열 수 없습니다" 또는 "개발자를 확인할 수 없습니다" 라는 메시지가 나타날 경우:**
- 설치된 앱은 Ad-hoc 서명으로 구성되어 있어 macOS의 Gatekeeper 시스템이 실행을 차단할 수 있습니다.
- 이런 경우 `Terminal` 앱을 열어 아래 명령어를 입력하여 격리 속성(quarantine)을 해제해주세요:
  ```bash
  xattr -cr /Applications/BindManager.app
  ```

**Docker 컨테이너 정보나 Shell/Log 기능이 작동하지 않는 경우:**
- 시스템에 Docker Desktop 또는 OrbStack과 같은 Docker 엔진이 백그라운드에서 정상적으로 실행 중인지 확인해주세요.
- 터미널에서 `docker ps` 명령어가 정상 작동하는지 확인합니다.
